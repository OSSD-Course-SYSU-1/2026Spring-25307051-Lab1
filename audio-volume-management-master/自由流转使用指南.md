# 自由流转功能使用指南

## 功能说明

本应用已实现基于分布式数据对象的自由流转功能，可以在两台虚拟机之间实时同步定时音量管理任务。

## 核心技术

使用HarmonyOS的**分布式数据对象（DistributedObject）**实现跨设备数据同步，无需系统级权限，适合虚拟机环境演示。

## 使用步骤

### 第一步：在设备A创建协同组

1. 打开应用，进入"定时音量管理"页面
2. 点击右上角设置按钮（齿轮图标）
3. 选择"自由流转设置"
4. 点击"创建新协同组"
5. 系统会生成一个SessionId，**请复制保存这个ID**

### 第二步：在设备B加入协同组

1. 在另一台虚拟机上打开应用
2. 进入"定时音量管理"页面
3. 点击右上角设置按钮
4. 选择"自由流转设置"
5. 在输入框中粘贴从设备A复制的SessionId
6. 点击"加入协同组"

### 第三步：分享任务

**在设备A上：**
1. 长按任务卡片进入多选模式
2. 选择要分享的任务（可多选）
3. 点击底部操作栏的"分享"按钮
4. 任务会自动同步到设备B

**在设备B上：**
- 设备B会自动接收任务并显示
- 无需任何操作

## 功能特点

### 1. 实时同步
- 任务数据实时同步到协同组内的所有设备
- 无需手动刷新

### 2. 双向通信
- 任一设备都可以分享任务
- 其他设备自动接收

### 3. 无需权限
- 使用分布式数据对象API
- 不需要系统级权限
- 适合虚拟机环境

### 4. 简单易用
- 只需SessionId即可建立连接
- 无需复杂的设备发现流程

## 技术实现

### 核心组件

1. **TaskDistributedManager** - 任务分布式管理器
   - 管理分布式数据对象
   - 处理任务数据的发布和接收
   - 维护协同组会话

2. **DistributedShareService** - 分布式分享服务
   - 集成TaskDistributedManager
   - 提供统一的分享接口
   - 管理会话生命周期

3. **SessionInputDialog** - 会话输入对话框
   - 提供创建/加入协同组的UI
   - 显示当前会话状态
   - 支持离开协同组

### 数据流程

```
设备A                          设备B
  |                              |
  | 1. 创建协同组                 |
  |    生成SessionId             |
  |                              |
  |                              | 2. 加入协同组
  |                              |    输入SessionId
  |                              |
  | 3. 选择任务并分享             |
  |    发布到分布式对象           |
  |                              |
  |                              | 4. 自动接收任务
  |                              |    保存到本地数据库
  |                              |    刷新UI显示
```

## 注意事项

### 1. 网络要求
- 两台设备必须在同一网络下
- 确保网络允许设备间通信（无AP隔离）

### 2. 账号要求
- 两台设备需要登录同一华为账号
- 在虚拟机设置中登录账号

### 3. SessionId管理
- SessionId是协同组的唯一标识
- 请妥善保存，避免泄露
- 每次创建协同组都会生成新的SessionId

### 4. 数据冲突
- 当前实现为覆盖式同步
- 接收方会直接保存接收到的任务
- 如需合并策略，可进一步优化

## 故障排查

### 问题1：无法创建协同组

**可能原因：**
- 分布式数据对象初始化失败
- 网络连接异常

**解决方案：**
- 检查网络连接
- 重启应用
- 查看日志输出

### 问题2：无法加入协同组

**可能原因：**
- SessionId输入错误
- 设备A未创建协同组
- 网络隔离

**解决方案：**
- 确认SessionId正确
- 确保设备A已创建协同组
- 检查网络设置

### 问题3：任务未同步

**可能原因：**
- 未加入同一协同组
- 分布式数据对象通信失败
- 数据序列化错误

**解决方案：**
- 确认两设备使用相同SessionId
- 查看日志中的错误信息
- 重新建立协同组

## 开发说明

### API使用

```typescript
// 创建分布式管理器
const manager = TaskDistributedManager.getInstance(context);

// 生成并设置SessionId
const sessionId = manager.generateSessionId();
manager.setSessionId(sessionId);

// 发布任务
manager.publishTasks(tasks);

// 接收任务
manager.onTaskSync((tasks: TimerTask[]) => {
  // 处理接收到的任务
});
```
