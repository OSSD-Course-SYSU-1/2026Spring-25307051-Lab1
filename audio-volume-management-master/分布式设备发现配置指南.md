# 分布式设备发现配置指南

## 问题原因

分布式设备发现找不到设备，通常有以下几个原因：

### 1. 权限不足

当前配置只有 `DISTRIBUTED_DATASYNC` 权限，但设备发现还需要其他权限。

### 2. 设备未登录同一华为账号

两台设备必须登录同一个华为账号才能进行分布式通信。

### 3. 设备未开启分布式功能

需要在设备设置中开启分布式功能。

### 4. 网络配置问题

虽然两台设备在同一网络，但可能存在防火墙或网络隔离。

## 解决方案

### 步骤 1：添加必要的权限

需要在 `module.json5` 中添加以下权限：

```json
{
  "name": "ohos.permission.ACCESS_SERVICE_DM",
  "reason": "$string:dm_permission_reason",
  "usedScene": {
    "abilities": ["EntryAbility"],
    "when": "inuse"
  }
}
```

**注意：** `ACCESS_SERVICE_DM` 是系统级权限，需要系统签名才能使用。

### 步骤 2：登录华为账号

1. 在真机上：设置 > 华为账号 > 登录
2. 在虚拟机上：设置 > 华为账号 > 登录
3. **两台设备必须登录同一个华为账号**

### 步骤 3：开启分布式功能

1. 在真机上：设置 > 更多连接 > 多设备协同 > 开启
2. 在虚拟机上：设置 > 更多连接 > 多设备协同 > 开启

### 步骤 4：检查网络

1. 确保两台设备在同一网络
2. 确保网络允许设备间通信（无AP隔离）
3. 确保防火墙不阻止分布式通信

### 步骤 5：重启应用

完成以上配置后，重启应用并再次尝试设备发现。

## 替代方案

如果无法使用分布式设备发现，可以考虑：

### 方案 1：使用二维码分享

1. 生成包含任务数据的二维码
2. 另一台设备扫描二维码获取数据

### 方案 2：使用云同步

1. 将任务数据上传到云端
2. 另一台设备从云端下载

### 方案 3：使用蓝牙/Wi-Fi直连

1. 使用蓝牙或Wi-Fi直连传输数据
2. 不依赖分布式功能

## 当前限制

由于 `ACCESS_SERVICE_DM` 是系统级权限，普通应用无法使用。因此：

1. **设备发现功能可能无法正常工作**
2. **建议使用替代方案（二维码、云同步等）**

## 测试步骤

1. 确保两台设备登录同一华为账号
2. 确保两台设备开启分布式功能
3. 确保两台设备在同一网络
4. 重启应用
5. 点击分享按钮
6. 查看是否能发现设备

如果仍然找不到设备，请检查日志输出，查看具体错误信息。
