# Audio Volume Management 新增功能详解

## 一、功能介绍

本项目在原有音频流音量管理基础上，新增了**定时任务管理系统**和**多端协同功能**，实现了智能化的音量自动调节和跨设备数据共享。

### 新增核心功能

#### 1. 定时任务管理
支持创建定时任务自动调节音量，包括：
- **单次任务**：指定日期时间执行一次，支持一次性（执行后删除）和重复利用（执行后禁用可再次启用）两种模式
- **每周循环**：每周指定星期几的固定时间执行
- **每月循环**：每月指定日期的固定时间执行
- **时间段锁定**：在指定时间段内锁定音量设置，防止被其他应用或用户修改

#### 2. 任务分组管理
支持将多个定时任务分组管理，便于批量操作和分类管理：
- 创建、编辑、删除任务分组
- 分组内任务的批量启用/禁用
- 分组级别的快速操作

#### 3. 多端协同与数据共享
实现跨设备的数据共享和任务同步：
- 分布式设备发现与连接
- 任务数据跨设备分享
- 多设备间的任务同步
- 自由流转支持（手机、平板、2in1设备）

#### 4. 音量监控与锁定
增强的音量控制能力：
- 实时监控音量变化
- 时间段内音量锁定，防止外部修改
- 音量变化事件监听与响应

#### 5. 任务执行历史
完整的任务执行记录系统：
- 记录每次任务执行的时间和结果
- 支持查看任务执行历史
- 失败任务的错误信息记录

#### 6. 多选与批量操作
增强的UI交互能力：
- 任务多选功能
- 批量删除、启用、禁用任务
- 多选工具栏快速操作

**关键方法实现**：

```typescript
// 检查当前是否在锁定时间段内（支持跨天情况）
isInLockPeriod(): boolean {
  const now = new Date();
  const currentTime = now.getHours() * 60 + now.getMinutes();
  const startTime = this.hour * 60 + this.minute;
  const endTime = this.endHour! * 60 + this.endMinute!;
  
  // 处理跨天情况（例如：22:00 - 06:00）
  if (startTime > endTime) {
    return currentTime >= startTime || currentTime <= endTime;
  } else {
    return currentTime >= startTime && currentTime <= endTime;
  }
}

// 计算下一次执行时间
getNextExecuteTime(): number {
  const now = new Date();
  
  if (this.timeMode === TimeMode.WEEKLY) {
    // 计算下一个指定星期几的时间
    const targetDay = this.dayOfWeek!;
    const currentDay = now.getDay() === 0 ? 7 : now.getDay();
    
    let daysUntil = targetDay - currentDay;
    if (daysUntil < 0 || (daysUntil === 0 && this.isTimePassed(now))) {
      daysUntil += 7;
    }
    
    const nextDate = new Date(now);
    nextDate.setDate(now.getDate() + daysUntil);
    nextDate.setHours(this.hour, this.minute, 0, 0);
    return nextDate.getTime();
  }
  // ... 其他模式的计算逻辑
}
```

### 2. 定时调度服务

调度服务负责任务的调度和执行，使用setTimeout实现定时触发：

```typescript
export class TimerScheduler implements ITimerScheduler {
  private scheduledTasks: Map<string, number> = new Map(); // taskId -> timerId
  
  // 调度任务
  scheduleTask(task: TimerTask): void {
    // 取消已存在的调度
    this.cancelTask(task.id);
    
    // 计算下一次执行时间
    const nextExecuteTime = this.calculateNextExecuteTime(task);
    const delay = nextExecuteTime - Date.now();
    
    if (delay <= 0) {
      Logger.warn(TAG, `Task ${task.id} execution time has passed`);
      return;
    }
    
    // 使用setTimeout调度任务
    const timerId = setTimeout(() => {
      this.executeTask(task);
    }, delay);
    
    this.scheduledTasks.set(task.id, timerId);
  }
  
  // 执行任务
  async executeTask(task: TimerTask): Promise<void> {
    // 检查任务状态
    const currentTask = await this.taskService.getTaskById(task.id);
    if (!currentTask || currentTask.status !== TaskStatus.ENABLED) {
      return;
    }
    
    // 根据任务类型执行不同操作
    if (task.taskType === TaskType.PERIOD) {
      // 时间段锁定任务
      await this.periodLockManager.startPeriodLock(task);
    } else {
      // 瞬时任务：立即设置音量
      this.volumeService.setVolumes(task.volumeSettings);
    }
    
    // 记录执行结果
    const execution = new TaskExecutionModel({
      taskId: task.id,
      executeTime: Date.now(),
      result: 'SUCCESS'
    });
    await this.taskService.recordExecution(execution);
    
    // 循环任务重新调度
    if (task.timeMode !== TimeMode.ONCE) {
      this.scheduleTask(task);
    }
  }
}
```

### 3. 时间段锁定管理

时间段锁定功能在指定时间段内锁定音量，防止被外部修改：

```typescript
export class PeriodLockManager {
  private volumeMonitor: VolumeMonitor;
  private isLocked: boolean = false;
  private lockTask: TimerTask | null = null;
  
  // 启动时间段锁定
  async startPeriodLock(task: TimerTask): Promise<void> {
    this.lockTask = task;
    this.isLocked = true;
    
    // 设置初始音量
    this.volumeService.setVolumes(task.volumeSettings);
    
    // 启动音量监控，拦截外部音量修改
    this.volumeMonitor.startMonitoring((volumeType, newVolume) => {
      if (this.isLocked && task.isInLockPeriod()) {
        // 在锁定时间段内，恢复到锁定音量
        const setting = task.volumeSettings.find(s => s.volumeType === volumeType);
        if (setting && !setting.mute) {
          this.volumeService.setVolume(volumeType, setting.volumeValue);
        }
      }
    });
    
    // 计算锁定结束时间并设置定时器
    const lockEndTime = task.getLockEndTime();
    const delay = lockEndTime - Date.now();
    
    setTimeout(() => {
      this.stopPeriodLock();
    }, delay);
  }
  
  // 停止时间段锁定
  stopPeriodLock(): void {
    this.isLocked = false;
    this.volumeMonitor.stopMonitoring();
    this.lockTask = null;
  }
}
```

### 5. 任务数据库

使用RDB（Relational Database）存储任务和执行记录：

```typescript
export class TimerTaskDatabase {
  private rdbStore: relationalStore.RdbStore | null = null;
  
  // 初始化数据库
  async initialize(): Promise<void> {
    const config: relationalStore.StoreConfig = {
      name: 'TimerTask.db',
      securityLevel: relationalStore.SecurityLevel.S1
    };
    
    this.rdbStore = await relationalStore.getRdbStore(this.context, config);
    
    // 创建任务表
    const createTaskTableSql = `
      CREATE TABLE IF NOT EXISTS timer_tasks (
        id TEXT PRIMARY KEY,
        name TEXT,
        timeMode TEXT,
        taskType TEXT,
        hour INTEGER,
        minute INTEGER,
        volumeSettings TEXT,
        status TEXT,
        createdTime INTEGER,
        lastExecuteTime INTEGER
      )
    `;
    await this.rdbStore.executeSql(createTaskTableSql);
    
    // 创建执行记录表
    const createExecutionTableSql = `
      CREATE TABLE IF NOT EXISTS task_executions (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        taskId TEXT,
        executeTime INTEGER,
        result TEXT,
        errorMessage TEXT
      )
    `;
    await this.rdbStore.executeSql(createExecutionTableSql);
  }
  
  // 保存任务
  async insertTask(task: TimerTask): Promise<void> {
    const valueBucket: relationalStore.ValuesBucket = {
      id: task.id,
      name: task.name,
      timeMode: task.timeMode,
      taskType: task.taskType,
      hour: task.hour,
      minute: task.minute,
      volumeSettings: JSON.stringify(task.volumeSettings),
      status: task.status,
      createdTime: task.createdTime
    };
    await this.rdbStore!.insert('timer_tasks', valueBucket);
  }
}
```

### 8. 架构设计特点

**三层架构**：
- **Application层**：UI组件和页面，负责用户交互
- **Communication层**：音频通信控制，负责音频播放和音量控制
- **Framework层**：核心业务逻辑和服务，负责定时任务、数据持久化、多端协同等

**MVVM模式**：
- View：页面和组件
- ViewModel：TimerTaskViewModel、TaskGroupViewModel
- Model：TimerTask、TaskGroup、TaskExecution等数据模型

**服务化设计**：
- 每个功能模块通过Service接口解耦
- 使用依赖注入方式组装服务
- 便于单元测试和模块替换

**数据持久化**：
- 使用RDB存储任务和执行记录
- 支持数据迁移和版本管理
- 提供完整的CRUD操作接口

**权限管理**：
- 音频设置权限：`ohos.permission.MODIFY_AUDIO_SETTINGS`
- 分布式数据同步权限：`ohos.permission.DISTRIBUTED_DATASYNC`
- 设备管理权限：`ohos.permission.ACCESS_SERVICE_DM`

**多设备支持**：
- 通过DeviceAdapter适配不同设备
- 使用CapabilityManager检测设备能力
- 支持手机、平板、2in1设备
